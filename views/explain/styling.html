Styling

CSS is a powerful system to manage style of your application. Infortunately, in most of the cases it mixes 2 usages :

  1. Theming : colors, background images, fonts... which is most of the time a definition global to the whole application
  2. Layout and positioning : How the elements are displayed (all the float, position: absolute, flex, ...). Which is usually closely linked to the HTML markup to which it is related
VeloxJS view system considers that the first point is outside of the view scope : just keep your global CSS file (created with with whatever CSS pre-processor you like if you are kind of this kind of tools).

For the second point, we think that the CSS rules should stay as near the HTML as possible as it is often closely linked to it

So you can just put them right on the top of your HTML view file like this : 

todo_app/views/welcome.html




    (...)

There is absolutely nothing special with this CSS code, it is just normal plain CSS rules that will be loaded when you will open the view the first time

In the case where you have a complicated view with a lot of markup and CSS rules, you can put CSS rules in a separated file and reference it in your view like this : 

todo_app/views/welcome.html



        (...)

welcome.css is a CSS file in the same directory as the view HTML file. You may use relative path if you need to reference a file elsewhere.
